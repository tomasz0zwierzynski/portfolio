Integracja systemów IT za pomocą szyny ESB

Java 8
Czerwiec 2020 - Obecnie

Projekt dotyczy zintegrowania ze sobą wielu aplikacji, systemów i baz danych ze sobą. Przyczyną konieczności integracji jest przejście ze starszych systemów na nowsze, mające większe możliwości automatyzacji procesów realizowanych przez klienta. Oprócz dostarczenia nowego, gotowego rozwiązania, wymagane jest także przeprowadzenie migracji dużej ilości danych, co w założeniu wymaga równoległego funkcjonowania wszystkich systemów. Do zrealizowania integracji wykorzystana została szyna danych WSO2.

Zadania związane były z rozpoznawaniem API integrowanych systemów i tworzeniem sekwencji obsługujących poszczególne metody HTTP, tworzeniem API szyny danych na podstawie dokumentacji projektowej oraz pisanem custom mediatorów w Javie. Szyna oprócz przetwarzania komunikatów, miała za zadanie wykonywać cyklicznie inne zadania polegające na wywoływaniu zapytań do bazy danych.
