Program testujący aplikację kliencką systemu zarządzania ruchem

Java 8
Październik 2019 - Grudzień 2019

Istniejący projekt automatyzacji zarządzania ruchem składający się z wielu podsystemów, wymagał istnienia monitorowania działania wielu aplikacji klienckich z punktu widzenia użytkownika, ze względu na warunki SLA. Wybrano rozwiązanie polegające na cyklicznym uruchamianiu aplikacji, która automatycznie przechodzi zdefiniowane ścieżki w aplikacji klienckiej takie jak logowanie, wczytanie odpowiedniej tabeli, przechodzenie do ustawień itp. Takie cykliczne pomiary dostępności miały być następnie zapisywane do bazy danych InfluxDB.

W tym celu napisano aplikację Java z wykorzystaniem Selenium - narzędzia automatyzującego przeglądarkę oraz sterownik przeglądarki chrome. Utworzyłem mechanizm, który umożliwia wykonanie dowolnej sekwencji kroków zdefiniowanych w pliku json. Aplikacja uruchamiana jest cyklicznie za pomocą Crona w systemie Linux, wykonuje pomiar i zapisuje próbkę w skonfigurowanej bazie danych.
